# AI疗愈助手 PRD
## 大学生焦虑情绪疏导产品

---

## 📋 产品概述

### 基本信息
- **产品名称**: AI疗愈助手
- **目标用户**: 20岁左右大学生群体（社交压力大、焦虑和空虚感强烈）
- **核心问题**: 日常焦虑情绪疏导 + 空虚感缓解
- **产品定位**: AI疗愈助手（帮用户变得更好，非恋爱/角色扮演）

### 市场背景
基于 AI 情感陪伴赛道市场调研的核心发现：
- **留存是行业最大痛点**: 多数用户1-2个月即流失，超半数月使用不足5天
- **记忆系统是产品分水岭**: 能"记住我"的产品用户粘性显著更高
- **重度使用可能加剧孤独**: MIT研究显示高频用户孤独感反而更强
- **年轻女性是核心付费人群**: 情感连接需求真实且付费意愿已被验证
- **通用陪伴产品正在死掉**: 行业共识是必须做垂直细分才能存活
- **下一代产品需要主动性**: 从"用户不问AI不答"进化到主动关怀

### 产品方向
- **核心理念**: 帮助用户变得更好
- **核心方法**: AI疗愈对话 + 微行动建议 + 情绪数据可视化
- **差异化定位**: 专注于大学生焦虑场景，垂直细分赛道

## 🎯 MVP 核心功能

### 功能1: AI情绪签到对话 🗣️

**核心流程**（3-5分钟结构化对话）：
```
开场 → 情绪状态探索 → 深度挖掘 → 结束与建议
```

#### 详细对话设计

**📍 开场阶段**（30秒）
- **问题1**: "今天感觉怎么样？用1-2个词形容一下"
- **目的**: 快速建立连接，让用户轻松开口
- **设计要求**: 避免开放式过大，给出几个示例词（平静/焦虑/低落/疲惫/开心）

**📍 情绪探索阶段**（2-3分钟）
- **问题2**: "刚才说你是[用户回答的词]，是因为发生了什么吗？"
- **问题3**: "这种感觉是从什么时候开始的？今天还是之前就有了？"
- **问题4**: "这个情绪对你今天的影响大吗？（影响很大/一般/基本不影响）"
- **设计要求**:
  - 问题要有递进性，从现象到原因
  - 选择题 + 自由回答结合，降低用户压力
  - AI 回应要体现"记住我"（即使技术上暂时做不到长期记忆）

**📍 深度挖掘阶段**（1-2分钟）
- **问题5**: "面对这种感觉，你今天最需要什么？（倾听/建议/具体的行动）"
- **问题6**: "有没有什么特别让你困扰或者纠结的事情？"
- **设计要求**:
  - 针对焦虑场景，关注具体触发点
  - 为微行动建议做准备

**📍 结束与建议阶段**（30秒）
- **问题7**: "谢谢你愿意分享。根据你今天的情况，我有一个小建议..."
- **转入微行动建议**

---

### 功能2: 微行动建议 🎯

#### 行动类型库（根据焦虑/空虚场景）

**情绪调节类**
- 💡 **3分钟呼吸练习**: 4-7-8呼吸法，专注呼吸节奏
- 🎵 **听一首疗愈音乐**: 推荐平静风格的3-5分钟音乐
- ☕ **喝一杯温水**: 慢下来，感受温度和味道

**思维转换类**
- 📝 **写下三件值得感恩的事**: 转移注意力到积极面
- 🚶 **到阳台散步5分钟**: 换个环境，转换心情
- 🌅 **拍一张窗外的照片**: 关注生活中的小美好

**行动导向类**
- 📚 **读5页轻松的书**: 转移注意力，减少焦虑循环
- 🤝 **主动问候一个朋友**: 建立真实连接，缓解孤独
- 🏃 **做10个简单的拉伸动作**: 身体放松带动心理放松

#### 推荐逻辑
- **焦虑主导** → 呼吸练习/行动导向
- **空虚主导** → 思维转换/行动导向
- **低落主导** → 情绪调节/感恩练习
- **平静状态** → 维护性行动（散步/听音乐）

#### 行动设计原则
- ⏱️ 时间控制在3-10分钟
- 📋 步骤清晰，无需额外准备
- 🎯 具体可衡量（如"5分钟"而非"一会儿"）
- 💪 难度适中，避免挫败感

### 功能3: 情绪数据记录与可视化 📊

#### 数据采集维度

**基础指标**（每次签到必记）:
- 📅 日期时间
- 😊 情绪状态（1-7分制：1=非常低落，7=非常开心）
- 😟 情绪类型（焦虑/平静/低落/疲惫/开心 - 自动分类）
- 💭 关键词（用户自定义1-2个词）
- ⚡ 情绪强度（1-5分制：影响程度）

**扩展指标**（对话中提取）:
- 🔍 触发因素（学习压力/人际关系/自我评价/未知等）
- ⏰ 持续时间（今天/最近几天/一直）
- 🎯 采取的行动（后续反馈可加）

#### 可视化设计要求

**整体风格**:
- 柔和曲线，非锐利折线
- 低饱和配色，符合大地色/奶茶色
- 留白充足，不拥挤

**具体图表**:

**📈 7天情绪趋势图**
- X轴：日期（周一至周日）
- Y轴：情绪分数（1-7分）
- 曲线：柔和贝塞尔曲线
- 色彩：根据情绪值渐变（低分用暖色，高分用冷色）
- 交互：点击显示当天详情

**📊 30天情绪分布图**
- 类型：小提琴图或平滑直方图
- 显示情绪集中区间和波动范围
- 色彩：单色透明度变化
- 交互：显示某段情绪天数

**🏷️ 情绪类型占比饼图**
- 类型：环形饼图，非传统实心饼图
- 色彩：低饱和度大地色系
- 交互：点击查看该类型下的所有记录

**🗓️ 日历热力图**
- 样式：类似 GitHub contribution graph
- 但色彩更柔和，采用低饱和渐变
- 交互：点击某天查看详情

#### 数据展示原则
- 🎯 一屏能看懂：不复杂，不技术化
- 🌊 流动感：平滑过渡，不要跳跃式变化
- 🎨 情绪化设计：颜色、曲线反映情绪本身
- 📱 响应式：适配不同屏幕尺寸

## 🎨 设计风格要求

### 整体风格
- **风格参考**: Calm App 的留白感 + flomo 的轻量记录感
- **核心理念**: 疗愈、轻盈、有呼吸感

### 绝对避免的设计
- ❌ 暗色系
- ❌ 赛博风
- ❌ 二次元风格
- ❌ 过度渐变

### 色彩方案
**低饱和暖色调**:
- 🟤 大地色（米色、浅棕）
- 🧋 奶茶色（浅米白、淡茶色）
- 🌫️ 雾霾蓝（柔和蓝灰色）
- ❌ 避免粉红粉紫

### 对话界面设计
**气泡设计**:
- 📐 圆角半径适中（16-20px），不过圆也不过方
- 🎨 色彩：用户气泡用浅奶茶色（#F5F0E8），AI气泡用雾霾蓝（#E8F0F2）
- 📏 间距：气泡之间12-16px间距，对话留白充足
- 🌡️ 阴影：轻微阴影（0 2px 8px rgba(0,0,0,0.08）），不要扁平
- ⏱️ 时间戳：右下角小字，颜色极淡（#A0A0A0）

**文字排版**:
- 📝 字体：系统默认无衬线字体，避免过于设计化
- 🔤 字号：正文16px，时间戳12px
- 📏 行高：1.6-1.8，让对话有呼吸感
- 🔠 最多3-4行气泡，超过折叠

**输入区域**:
- 🎯 保持最小化，只留必要输入框
- 🌟 提示文字温柔："轻轻告诉我，今天感觉怎么样..."
- 📐 圆角设计与气泡一致
- ⏰ 不强制打字，提供快捷回复选项

## 🔄 用户使用流程

### 首次使用流程（新用户引导）
```
下载App → 首次对话引导（1分钟介绍） → 第一次情绪签到 → 查看情绪记录 → 明天提醒
```

### 日常使用流程（核心循环）
```
打开App → 开始情绪签到（3-5分钟） → AI给出微行动 → 完成行动（可选） → 查看情绪趋势 → 关闭App
```

### 异常处理流程
- **超过3天未签到** → 温柔提醒："好久不见，今天感觉怎么样？"
- **连续7天未签到** → 主动关怀："最近还好吗？如果愿意的话，聊聊吧"
- **检测到极端情绪关键词** → 温和建议："听起来你现在很难过，如果需要，我可以给你一些额外的建议"

## 🏗️ 技术架构概览

### 核心技术栈（vibe coding友好）
- **前端**: React + Tailwind CSS（快速实现设计要求）
- **AI对话**: Claude API（情绪理解 + 对话生成）
- **数据存储**: Supabase（用户数据 + 情绪记录）
- **图表可视化**: Recharts（符合"不Excel"要求的轻量图表库）

### 核心模块
1. **对话引擎**: 基于 Claude API 的情绪理解和对话生成
2. **行动推荐**: 基于情绪类型的简单规则匹配
3. **数据可视化**: 情绪数据的采集、存储和展示
4. **用户管理**: 基本认证和个人数据管理

### 关键集成
- Claude API：AI对话核心
- Supabase：数据持久化
- Recharts：情绪曲线可视化

## ⚠️ 风险与挑战

### 产品风险

**1. 留存问题**
- **风险**: 行业数据显示纯陪伴产品留存困难
- **缓解策略**: 通过微行动和情绪可视化建立正向循环
- **监控指标**: 7日/30日留存率

**2. 情感依赖风险**
- **风险**: MIT研究显示高频使用可能加剧孤独
- **缓解策略**: 在对话中适时提醒适度使用
- **监控指标**: 用户日均使用时长

**3. AI幻觉风险**
- **风险**: 对话可能给出不当建议
- **缓解策略**: 限制AI建议范围，仅限于日常情绪调节
- **免责声明**: 明确不是专业心理治疗

### 技术风险

**1. API成本控制**
- **风险**: Claude API 使用成本
- **缓解策略**: 优化对话长度，限制每日对话次数
- **监控指标**: 单用户日均Token消耗

**2. 数据隐私安全**
- **风险**: 用户情绪数据敏感性
- **缓解策略**: 端到端加密，数据本地化存储
- **合规要求**: 符合GDPR等隐私法规

### 市场风险

**1. 竞品快速跟进**
- **风险**: 产品模式容易被复制
- **缓解策略**: 通过优质用户体验建立品牌壁垒
- **差异化优势**: 专注于大学生焦虑场景的专业性

## 🚀 MVP 范围确认清单

### ✅ 核心功能（第一版必须）
1. **AI情绪签到对话**
   - 3-5分钟结构化对话
   - 理解用户当前情绪状态
   - 提供基础的情感支持

2. **微行动建议**
   - 根据情绪状态给出1-2个具体行动建议
   - 简单、可执行、有时间要求（如"5分钟呼吸练习"）
   - 建立正向反馈循环

3. **情绪数据记录与可视化**
   - 记录每次签到的基础情绪数据
   - 显示近7天/30天的情绪趋势曲线
   - 让用户看到自己的变化

### 📌 后续迭代方向（不做但有价值）
1. **跨会话记忆**（P2优先级） - 符合调研中"记住我"的关键洞察
2. **个性化签到提醒**（P3优先级） - 主动关怀的一部分
3. **情绪标签分类**（P4优先级） - 数据结构化有助于分析

### ❌ MVP不做
- 复杂的任务系统
- 社交功能
- 多种AI角色
- 高级付费功能

## 📊 核心逻辑闭环

```
用户开始签到 → AI引导对话 → 识别情绪状态 → 建议微行动 → 记录情绪数据 → 显示趋势曲线 → 用户感受到进步 → 明天继续签到
```

**这个闭环能够**:
- ✅ 解决焦虑和空虚感（核心问题）
- ✅ 建立使用习惯（留存基础）
- ✅ 提供正向反馈（趋势可视化）
- ✅ 体现产品价值（情绪改善）
